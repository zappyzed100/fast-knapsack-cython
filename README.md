# fast-knapsack-cython
## 100秒の壁を突破する：実務制約付きナップサック問題の高速化

汎用的な数理最適化ソルバー（Cbc/CP-SAT）では100秒以上を要する、複雑な実務制約を含むナップサック問題を、CythonとNumbaによる独自アルゴリズム実装で**3,000倍以上高速化**したプロジェクトです。

## 1. 解決する課題：実務制約の複雑性と規模
本プロジェクトでは、単なる「価値と重さ」の関係を超えた、以下の具体的な数値規模を持つ実務的な最適化問題を対象としています。

- **アイテム総数**: **2,000個**
- **カテゴリー（グループ）数**: **200種類**
- **多次元リソース制限（3種）**: 重さ・体積・電力等に見立てた3つの独立したコスト。
- **カテゴリー間排他制約（40組）**: 「グループAを選んだらグループBは不可」という強固な制約。
- **非線形セットボーナス**: 同一グループ内での選択数（3〜5個）に応じた加点。

## 2. 問題の計算困難性：2^2000 の壁
本問題の難しさは、膨大な探索空間と「解の分断」にあります。
- **天文学的な組み合わせ**: 2,000個のアイテムの組み合わせは $2^{2000}$ 通りを超えます。
- **高密度な排他制約**: 40組の排他ルールが解空間を分断し、汎用ソルバーの探索を停滞させます。
- **実測データ**: 厳密解を求める汎用ソルバー（Cbc）では、最適解の導出に**約100秒**を要します。

## 4. 高速化の技術アプローチ：戦略的パラダイムシフト

### ① 「証明」を捨てて「高速探索」に特化する
汎用ソルバーの役割は「最高値であることの数学的証明」ですが、本実装ではその証明コストを切り捨て、実務的な即時性を優先しました。
- **戦略**: 厳密解法ではなく、高速な**焼きなまし法（SA）**および**ハイブリッド進化計算（GA+SA）**を採用。
- **結果**: 100秒かけて「100点の証明」を待つ代わりに、**0.03秒**で実用的な解を得るパフォーマンスを実現しました。

### ② 「数式」を「ビット演算」に変換
汎用ソルバーにとって排他制約は巨大な行列計算の対象ですが、本実装ではこれを低レイヤーの処理に落とし込みました。
- **独自実装**: 40組の排他ルールを**ビットマスク**（`unsigned long long[16]`）に変換。
- **効率**: 違反チェックをCPUの1クロックで済む**ビット論理積（`&`）**に置換。判定コストを物理限界まで引き下げました。

### ③ 「Swap戦略」による局所解の突破
制約が厳しい問題では、1つのアイテムを入れ替えるだけでは解が改善しない「袋小路」に陥ります。
- **独自実装**: アイテムの追加時に重量制限を超えた場合、既存のアイテムを確率的に捨てて入れ替える**Swapロジック**を実装。
- **結果**: 汎用ソルバー（CP-SAT）が100秒かけて到達するスコアを、わずか数秒の進化計算で上回る効率を達成しました。

### ④ 並列進化計算による精度向上
- **マルチコア活用**: `prange`（Cython）や `parallel=True`（Numba）を用いて、複数の個体を並列にSAで磨き上げるハイブリッド構成。
- **エリート保存と交叉**: 優秀な個体を残しつつ、密度の高いアイテムを優先する**貪欲交叉（Greedy Crossover）**によって解の多様性と質を維持。

## 5. 環境構築
`uv` を用いて、Python 実行環境とビルドプロセスが管理されています。

```bash
# 環境の同期
uv sync
# データの生成
uv run python scripts/generate_and_save_problem.py
# Cythonのビルド
uv run python solver_cython/setup.py build_ext --inplace
# レポートの生成（各ソルバーの実行後に実施）
uv run python generate_report.py
