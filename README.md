# fast-knapsack-cython
## 125秒の壁を突破する：実務制約付きナップサック問題の高速化

汎用的な数理最適化ソルバー（PuLP/CBC）では現実的な時間（約125秒）を要する、複雑な実務制約を含むナップサック問題を、Cythonによる独自アルゴリズム実装で**2,000倍以上高速化**したプロジェクトです。

## 1. 解決する課題：実務制約の複雑性と規模
本プロジェクトでは、単なる「価値と重さ」の関係を超えた、以下の具体的な数値規模を持つ実務的な最適化問題を対象としています。

- **アイテム総数**: **2,000個**
- **カテゴリー（グループ）数**: **200種類**
- **多次元リソース制限（3種）**: 重さ・体積・電力等に見立てた3つの独立したコスト。
- **カテゴリー間排他制約（40組）**: 「グループAを選んだらグループBは不可」という強固な制約。
- **非線形セットボーナス**: 同一グループ内での選択数に応じた加点。

## 2. 問題の計算困難性：2^2000 の壁
本問題の難しさは、膨大な探索空間と「解の分断」にあります。
- **天文学的な組み合わせ**: 2,000個のアイテムの組み合わせは $2^{2000}$ 通りを超えます。
- **高密度な排他制約**: 40組の排他ルールが解空間を分断し、汎用ソルバーの分枝限定法を停滞させます。
- **実測データ**: 汎用ソルバー（PuLP/CBC）では、この複雑な制約網を解釈・計算するために**125秒以上**を要します。

## 4. 高速化の技術アプローチ：戦略的パラダイムシフト

### ① 「証明」を捨てて「探索」に特化する
汎用ソルバーの役割は「最高値であることの数学的証明」ですが、本実装ではその証明コストを切り捨てました。
- **戦略**: 厳密解に拘泥する分枝限定法ではなく、高速な近傍探索（山登り法）を採用。
- **結果**: 125秒かけて「100点満点の証明」を待つ代わりに、0.05秒で「99点の解」を出す実務的な即時性を実現しました。


### ② 「数式」を「ビット演算」に変換
汎用ソルバーにとって排他制約は「巨大な行列計算」の対象ですが、本実装ではこれを低レイヤーの処理に落とし込みました。
- **独自実装**: 40組の排他ルールを**256bitのビットマスク**（`unsigned long long[4]`）に変換。
- **効率**: ソルバーが行列演算で行う違反チェックを、CPUの1クロックで済む**ビット論理積（`&`）**に置換。判定コストを物理限界まで引き下げました。

### ③ 「2-for-1スワップ」による局所解の突破
制約が厳しい問題では、1つのアイテムを入れ替えるだけでは解が改善しない「袋小路」に陥ります。
- **独自実装**: 1つ追加するために邪魔な2つを確率的に捨てる「2-for-1スワップ」を実装。これにより、汎用ソルバーが停滞する複雑な制約網を飛び越え、より高いスコアへ到達可能にしました。


### ④ キャッシュ効率とゼロコピーの徹底
- **Memoryview**: NumPy配列の生データへ直接アクセスし、Pythonのオーバーヘッドを排除。
- **乱数の一括事前生成**: ループ内での生成を避け、キャッシュ効率を最大化。
